String

简介

  • Redis最基本的数据类型，一个key对应一个value
  • String类型是二进制安全的，即string可以包含任何数据，例如图片，视频或者序列化的对象
  • 一个Redis的string类型的value最多可以是512M

常用命令

设置/取出key的值

127.0.0.1:6379> set k1 v100
OK
127.0.0.1:6379> set k2 v100
OK
127.0.0.1:6379> get k1
"v100"
#覆盖
127.0.0.1:6379> set k1 v101

追加原始值

127.0.0.1:6379> APPEND k1 11
(integer) 6
127.0.0.1:6379> get k1
"v10011"

查询某个key所对应的value的长度

127.0.0.1:6379> STRLEN k1
(integer) 6

添加没有的k-v

127.0.0.1:6379> SETNX k1
(error) ERR wrong number of arguments for 'setnx' command
127.0.0.1:6379> setnx k3 100
(integer) 1

自增

#增量为1
127.0.0.1:6379> INCR k3
(integer) 101
#设置步长
127.0.0.1:6379> INCRBY k3 10
(integer) 110

自减

#减量为1
127.0.0.1:6379> decr k3
(integer) 100
#设置步长
127.0.0.1:6379> DECRBY k3 10
(integer) 100

Redis单命令的原子性主要得益于Redis的单线程，Redis使用单线程+多路I/O复用。

java中的i++操作不是原子性，当i=0时，两个线程分别对i进行++操作100次，所得的值处于2~200之间。

同时设置一个或多个k-v

127.0.0.1:6379> mset k1 v1 k2 v2 k3 v3
OK

同时获取一个或多个value

127.0.0.1:6379> mget k1 k2
1) "v1"
2) "v2"

同时设置一个或多个k-v，当且仅当给定key不存在

127.0.0.1:6379> msetnx k10 v10 k11 v11
(integer) 1
#失败的话不会有k-v被添加，此操作具有原子性

截取[v1,v2]之间的值，类似substring

127.0.0.1:6379> set name liangxi
OK
127.0.0.1:6379> GETRANGE name 0 4
"liang"

覆盖从起始位置开始的值

127.0.0.1:6379> SETRANGE name 0 liu
(integer) 7
127.0.0.1:6379> get name
"liungxi"

在添加新的k-v时设置过期时间

127.0.0.1:6379> SETEX temp 20 100  #过期时间是20s
OK

以新换旧，同时返回旧值

127.0.0.1:6379> GETSET name liangxi
"liungxi" 		#旧值
127.0.0.1:6379> get name
"liangxi"

数据结构

底层为简单动态字符串，是可以修改的字符串，内部结构实现类似于Java的ArrayList，采用预分配冗余空间的方式来减少内存的频繁分配。当字符串长度小于1M时，扩容是加倍现有的空间，当大于1M时，扩容只会在现有空间加1M，字符串最大长度为512M。

List

简介

  • 单键多值
  • 是简单的字符串列表，按照插入的顺序排序，可以将一个元素添加到头部或者尾部
  • 实际上是一个双向列表，对两端的操作性能高，但通过下标操作中间节点的性能较差

常用命令

从左边/右边插入一个或多个值

127.0.0.1:6379> LPUSH k1 v1 v2 v3
(integer) 3
127.0.0.1:6379> RPUSH k2 v1 v2 v3
(integer) 3

查询指定范围的元素

# 开始为0，结束为-1，意为查询所有元素
127.0.0.1:6379> LRANGE k1 0 -1
1) "v3"
2) "v2"
3) "v1"

查询key中元素的个数

127.0.0.1:6379> llen k1
(integer) 3

根据下标查询元素

127.0.0.1:6379> LINDEX k1 1
"v2"

linsert before/after 在的前面/后面插入

127.0.0.1:6379> LINSERT k1 before "v1" "v2"
(integer) 4
127.0.0.1:6379> LRANGE k1 0 -1
1) "v3"
2) "v2"
3) "v2"
4) "v1"

改变指定下标的值

127.0.0.1:6379> LSET k2 0 v10
OK

从左到右删除n个指定的value

127.0.0.1:6379> lrem  k1 2 "v2" 
(integer) 2

从左边或右边吐出一个值，值在键在，值光键亡

127.0.0.1:6379> LPOP k1
"v3"
127.0.0.1:6379> LRANGE k1 0 -1
1) "v1"
127.0.0.1:6379> RPOP k1
"v1"
127.0.0.1:6379> LRANGE k1 0 -1
(empty array)
127.0.0.1:6379> KEYS *
1) "k2"

从k1的右边吐出一个值，放在k2的左边

127.0.0.1:6379> LPUSH k1 v20 v21
(integer) 2
127.0.0.1:6379> RPOPLPUSH k1 k2
"v20"
127.0.0.1:6379> LRANGE k1 0 -1
1) "v21"

数据结构

List的数据结构为快速列表quickList。

在列表元素较少时，会使用一块连续的内存存储，这个结构式zipList，即压缩列表。

当数据量比较多的时候才会使用quickList。

因为普通的链表需要的附加指针空间太大，会造成大量浪费。Redis将链表和zipList结合起来，既满足快速的插入删除性能，又不会出现太大的空间浪费。

Set

简介

  • 提供与List类似的列表功能，但是不允许出现重复元素。
  • 可以判断元素是否在集合中。
  • 无序集合，底层是一个value为null的hash表，所以添加，删除，查找的复杂度都是O(1)。

常用命令

将一个或多个member加入到key中，已经存在的忽略

127.0.0.1:6379> sadd k1 v1 v2 v3 v3
(integer) 3

查询指定key中的元素

127.0.0.1:6379> SMEMBERS k1
1) "v2"
2) "v3"
3) "v1"

判断value是否在key中

127.0.0.1:6379> SISMEMBER k1 v1
(integer) 1

查询key中元素个数

127.0.0.1:6379> SCARD k1
(integer) 3

从key中随机吐出一个元素

127.0.0.1:6379> spop k1
"v2"

从key中随机取出n个元素，不会从集合中删除

127.0.0.1:6379> SRANDMEMBER k1 2
1) "v3"
2) "v1"

删除集合中的一个或多个元素

127.0.0.1:6379> srem  k1 v3 v1
(integer) 2

返回两个集合的交集元素

127.0.0.1:6379> SADD k1 v1 v2 v3
(integer) 3
127.0.0.1:6379> SADD k2 v2 v3 v4
(integer) 3
127.0.0.1:6379> SINTER k1 k2
1) "v2"
2) "v3"

返回两个集合的并集

127.0.0.1:6379> SUNION k1 k2
1) "v2"
2) "v4"
3) "v3"
4) "v1"

返回k1中有，而k2中没有的元素

127.0.0.1:6379> SDIFF k1 k2
1) "v1"

把value从k1移动到k2中

127.0.0.1:6379> SMOVE k1 k2 v1
(integer) 1
127.0.0.1:6379> SMEMBERS k1
1) "v2"
2) "v3"

数据结构

Set数据结构是dict字典，用哈希表实现。

Java中的HashSet的内部实现使用的是HashMap，只不过所有的value都指向同一个对象。

Redis的set结构也是一样的，内部也使用hash结构，所有的value都指向同一个内部值。

Hash

简介

Redis的Hash是一个键值对集合。

Hash的value部分是一个field和value的映射表，所以，hash特别适合用于存储对象。

类似于Java中的Map<String,Object>。

  • 方法一

    采用序列化操作，但要修改数据时，要先反序列化后在序列化，开销较大

  • 方法二

    重复存储用户id

  • 方法三

    最佳选择

常用命令

单个设置/取出数据

127.0.0.1:6379> HSET user01 id 1
(integer) 1
127.0.0.1:6379> HSET user01 name zhangsan
(integer) 1
127.0.0.1:6379> hget user01 id
"1"
127.0.0.1:6379> hget user01 name
"zhangsan"

批量设置hash数据，查询key中所有的field，value

127.0.0.1:6379> hmset user02 id 2 name lisi
OK
127.0.0.1:6379> HKEYS user02
1) "id"
2) "name"
127.0.0.1:6379> HVALS user02
1) "2"
2) "lisi"

查询key中是否存在指定的field

127.0.0.1:6379> HEXISTS user02 age
(integer) 0
127.0.0.1:6379> HEXISTS user02 id
(integer) 1

将哈希表 key 中的域 field 的值设置为 value ，当且仅当域 field 不存在

127.0.0.1:6379> HSETNX user02 age 18
(integer) 1

为哈希表 key 中的域 field 的值加上增量

127.0.0.1:6379> HINCRBY user02 age 2
(integer) 20

数据结构

Hash类型对应的数据结构是两种：ziplist（压缩列表），hashtable（哈希表）。当field-value长度较短且个 数较少时，使用ziplist，否则使用hashtable。

Zset

简介

Zset和Set类似，是一个没有重复元素的字符串集合。

Zset中的每个成员都关联了一个评分(score)，这个评分被用来排序，从低到高。集合中的成员是唯一的，但评分可以重复。

常用命令

将一个或多个 member 元素及其 score 值加入到有序集 key 当中

127.0.0.1:6379> zadd topn 100 c 200 c++ 300 java
(integer) 3

返回有序集 key 中，下标在之间的元素

127.0.0.1:6379> zrange topn 0 -1
1) "c"
2) "c++"
3) "java"
#带WITHSCORES，可以让分数一起和值返回到结果集。
127.0.0.1:6379> zrange topn 0 -1 withscores
1) "c"
2) "100"
3) "c++"
4) "200"
5) "java"
6) "300"

返回有序集 key 中，所有 score 值介于 min 和 max 之间(包括等于 min 或 max )的成员。有序集成员按 score 值递增(从小到大)次序排列

127.0.0.1:6379> zrangebyscore topn 101 300
1) "c++"
2) "java"
127.0.0.1:6379> zrangebyscore topn 101 300 withscores
1) "c++"
2) "200"
3) "java"
4) "300"
#同上，改为从大到小排列
127.0.0.1:6379> ZREVRANGEBYSCORE topn 300 100
1) "java"
2) "c++"
3) "c"

为元素的score加上增量

127.0.0.1:6379> ZINCRBY topn 10 java
"310"

统计该集合，分数区间内的元素个数

127.0.0.1:6379> ZCOUNT topn 100 300
(integer) 2

返回该值在集合中的排名，从0开始

127.0.0.1:6379> zrank topn c++
(integer) 1

删除该集合下，指定值的元素

127.0.0.1:6379> zrem  topn java 
(integer) 1
127.0.0.1:6379> ZRANGE topn 0 -1
1) "c"
2) "c++"

数据结构

SortedSet(zset)是Redis提供的一个非常特别的数据结构，一方面它等价于Java的数据结构Map<String, Double>，可以给每一个元素value赋予一个权重score，另一方面它又类似于TreeSet，内部的元素会按照权重score进行排序，可以得到每个元素的名次，还可以通过score的范围来获取元素的列表。

zset底层使用了两个数据结构

（1）hash，hash的作用就是关联元素value和权重score，保障元素value的唯一性，可以通过元素value找到相应的score值。

（2）跳跃表，跳跃表的目的在于给元素value排序，根据score的范围获取元素列表。
